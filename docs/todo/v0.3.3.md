- [x] 优化 server daemon 内存占用 ✅ 2026-04-27 20:39:00
    - 功能说明：
      - daemon 启动时内存占用约 61MB，主要因为 scheduler.py 从 executor.py 导入了重型依赖
      - executor → notifier → channels → httpx / lark-oapi / websockets 等库在 server 启动时全部被加载
    - 解决方案：
      - `scheduler.py` 独立定义 `LOGS_DIR` 和 `SYSTEM_LOG`，不再从 `executor` 导入
      - `run_task_with_notify` 改为任务触发时的延迟导入（lazy import）
      - server 启动时只加载 `storage.py` 和 `scheduler.py`
      - `executor.py` 及其重型依赖只在实际执行任务时才加载
    - 优化效果：
      - RSS 从 ~61MB 降至 ~21MB（降幅约 65%）
    - 修改内容：
      - `src/claw_cron/scheduler.py`：独立定义日志路径常量，延迟导入 run_task_with_notify
    - 提交记录：
      - `d78401b` perf(scheduler): lazy import executor to reduce daemon memory usage
