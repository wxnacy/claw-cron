- [x] 为 Task 增加 `cwd`（工作目录）支持 ✅ 2026-05-06 15:59:00
    - 需求：允许任务指定执行时的工作目录，使脚本中的相对路径（如 `./run.sh`、`python ./helper.py`）能正确解析
    - 功能说明/问题原因：
      - 原有实现：`subprocess.run()` 未传入 `cwd` 参数，子进程继承父进程当前工作目录
      - `claw-cron run`、前台 `server`、daemon `server --daemon`、brew service 四种启动方式的 CWD 各不相同
      - 同一任务在不同启动方式下，相对路径行为不一致
    - 解决方案：
      - Task dataclass 新增 `cwd: str | None = None` 字段
      - `subprocess.run()` 传入 `cwd=task.cwd`（command 和 agent 类型）
      - CLI `add` 和 `command` 命令新增 `--cwd` 选项
      - `list` 表格新增 CWD 列，未设置时显示 `-`
      - `temp_task` 构造时传递 `cwd=task.cwd`
    - 修改内容：
      - `src/claw_cron/storage.py`：Task 新增 cwd 字段
      - `src/claw_cron/executor.py`：subprocess.run() 传入 cwd，temp_task 传递 cwd
      - `src/claw_cron/cmd/add.py`：add 命令新增 --cwd 选项
      - `src/claw_cron/cmd/command.py`：command 命令新增 --cwd 选项，交互模式支持输入 cwd
      - `src/claw_cron/cmd/list.py`：list 表格新增 CWD 列
      - `AGENTS.md`：更新命令和参数说明
      - `README.md`：新增 --cwd 使用示例
    - 提交记录：

      - 9aa3817 docs: document cwd support in AGENTS.md and README.md
