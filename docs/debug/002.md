
## BUG FIX

- [x] 相同任务在 run 和后台执行时，表现不一样 ✅ 2026-05-06 15:10:00
    - `codebuddy-sign` 使用 `run` 运行可以正常执行
    - `codebuddy-sign` 在 `server` 中执行时报错：`Error: diary file not found: /04-周期总结/01-日记/2026-05-06.md`
    - 功能说明/问题原因：
      - 用户在脚本中使用了 `${OBSIDIAN_HOME}`，该变量定义在 `.zshrc` 中
      - `claw-cron run` 继承交互式 shell 环境，变量存在
      - `claw-cron server --daemon` 双 fork 为守护进程后，未经过 login shell 初始化，profile 变量缺失
      - `subprocess.run(..., env=env)` 完全替换了子进程环境，导致变量不可用
    - 解决方案：
      - 在 `executor.py` 新增 `_load_shell_env()` / `_get_shell_env()`，通过 `$SHELL -l -c env` 捕获 login shell 完整环境变量
      - 修改 `_build_env()` 合并顺序：`login_shell_env` → `os.environ` → `CLAW_*` 变量
      - 带全局缓存，只加载一次
    - 修改内容：
      - `src/claw_cron/executor.py`：新增 login shell 环境加载逻辑
