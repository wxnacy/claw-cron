
## BUG FIX

- [x] `chat -a codebuddy` 提问 `列出所有任务` 程序会卡住，`ctrl+c` 后报错 ✅ 2026-05-02 06:40:00
    - 功能说明/问题原因：
      - codebuddy provider 的 `chat_with_tools` 中传给 `query()` 的 `options` 是 dict，而 SDK 内部访问 `options.session_id`，导致 `'dict' object has no attribute 'session_id'` 错误
      - `_run_query` 中将 SDK 返回的 dataclass 消息当作 dict 解析，导致 `'SystemMessage' object has no attribute 'get'` 错误
      - MCP handler 返回假结果 `{"status": "tool_call_requested"}`，导致 SDK 内部模型陷入无限工具调用循环
      - `_run_query` 收集了所有 ToolUseBlock（包括内置工具 Bash/Skill/Read 等），导致 `_handle_tool` 输出大量 "Unknown tool"
      - codebuddy CLI 的权限系统默认拒绝工具调用，模型反复尝试，导致 `query()` 流持续很长时间
    - 解决方案：
      - 使用 `CodeBuddyAgentOptions` 对象代替 dict 作为 `options` 参数
      - 改用 `isinstance` 检查和 dataclass 属性访问解析消息
      - 在 MCP handler 中真正执行工具（通过 `_execute_tool`），避免假结果循环
      - 添加 `can_use_tool` 回调，只允许 `mcp__claw_cron_tools__*` 前缀的工具，阻止内置工具无限调用
      - `_run_query` 中设置 45 秒超时保护，防止 `query()` 流永不结束
      - `_run_query` 只收集文本内容，不再收集 tool_calls，由 SDK 内部处理工具调用
      - 在 `chat.py` 中为 codebuddy provider 添加关键词匹配快速路径，常见操作直接执行
    - 修改内容：
      - `src/claw_cron/providers/codebuddy.py`：修复 options 类型、消息解析、添加 `_execute_tool`、修复闭包、添加权限回调和超时
      - `src/claw_cron/cmd/chat.py`：添加 codebuddy provider 关键词匹配快速路径
    - 提交记录：
