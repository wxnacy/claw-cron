# Plan 04-03: Register server Command in CLI

## Objective

Update `src/claw_cron/cli.py` to import and register the `server` command, making `claw-cron server` and `claw-cron server --daemon` available.

## Context

<context>
- src/claw_cron/cli.py — existing Click group with add, list, delete, run, log, chat commands
- src/claw_cron/cmd/server.py — server command (created in Plan 04-02)
</context>

## Tasks

<task id="04-03-1" type="execute">
  <title>Add server import and registration to cli.py</title>

  <read_first>
    - src/claw_cron/cli.py (current imports and cli.add_command calls)
    - src/claw_cron/cmd/server.py (server command name)
  </read_first>

  <action>
In `src/claw_cron/cli.py`, add exactly these two changes:

1. Add import after the existing `from claw_cron.cmd.run import run` line:
   ```python
   from claw_cron.cmd.server import server
   ```

2. Add registration after the existing `cli.add_command(chat)` line:
   ```python
   cli.add_command(server)
   ```

The final cli.py should have these imports (in alphabetical order by module name):
```python
from claw_cron.cmd.add import add
from claw_cron.cmd.chat import chat
from claw_cron.cmd.delete import delete
from claw_cron.cmd.list import list_tasks
from claw_cron.cmd.log import log
from claw_cron.cmd.run import run
from claw_cron.cmd.server import server
```

And these registrations:
```python
cli.add_command(add)
cli.add_command(list_tasks)
cli.add_command(delete)
cli.add_command(run)
cli.add_command(log)
cli.add_command(chat)
cli.add_command(server)
```
  </action>

  <acceptance_criteria>
    - `grep "from claw_cron.cmd.server import server" src/claw_cron/cli.py` exits 0
    - `grep "cli.add_command(server)" src/claw_cron/cli.py` exits 0
    - `uv run claw-cron --help` output contains "server"
    - `uv run claw-cron server --help` exits 0 and output contains "--daemon"
  </acceptance_criteria>
</task>
