### Task 3: Implement channels add command

<task type="auto">

<action>
Add the `add` subcommand to channels group:

```python
@channels.command()
@click.option('--channel-type',
              type=click.Choice(['qqbot'], case_sensitive=False),
              prompt='Channel type',
              help='Type of channel to configure')
@click.option('--app-id',
              prompt='AppID',
              help='QQ Bot AppID from q.qq.com')
@click.option('--client-secret',
              prompt=True,
              hide_input=True,
              help='QQ Bot Client Secret')
@click.option('--capture-openid',
              is_flag=True,
              default=False,
              help='Connect WebSocket to capture user openid (Phase 2)')
def add(channel_type: str, app_id: str, client_secret: str, capture_openid: bool) -> None:
    """Add a new message channel configuration."""
```

**Implementation steps:**

1. **Validate credentials before saving:**
```python
with console.status("[bold green]Validating credentials..."):
    try:
        # Use QQBotChannel._get_access_token logic
        response = httpx.post(
            "https://bots.qq.com/app/getAppAccessToken",
            json={"appId": app_id, "clientSecret": client_secret},
            timeout=10.0
        )
        data = response.json()
        if data.get("code", 0) != 0:
            raise click.ClickException(f"Validation failed: {data.get('message')}")
    except httpx.RequestError as e:
        raise click.ClickException(f"Connection failed: {e}")
```

2. **Save to config.yaml:**
```python
config = load_config()
if "channels" not in config:
    config["channels"] = {}
config["channels"]["qqbot"] = {
    "app_id": app_id,
    "client_secret": client_secret,
    "enabled": True
}
save_config(config)
```

3. **Handle capture_openid flag:**
```python
if capture_openid:
    console.print("[yellow]Note: OpenID capture requires WebSocket connection (Phase 2)[/yellow]")
    console.print("[dim]Run 'claw-cron channels capture' after Phase 2 implementation[/dim]")
```

**Error handling:**
- Invalid credentials → show error, don't save
- Network error → show connection error
- File permission error → show helpful message

**Do NOT:**
- Save credentials without validation
- Store client_secret in plain text in logs
- Use synchronous HTTP in async context (use httpx sync for CLI commands)
