<action>
创建 `src/claw_cron/channels/wecom.py`，内容如下：

1. **WeComConfig**（继承 BaseSettings + ChannelConfig）：
   - `corp_id: str | None = Field(default=None)`
   - `agent_id: int | None = Field(default=None)`
   - `secret: str | None = Field(default=None)`
   - `class Config: env_prefix = "CLAW_CRON_WECOM_"; env_file = ".env"; extra = "ignore"`

2. **TokenInfo dataclass**（与 qqbot.py 相同结构）：
   - `access_token: str`
   - `expires_at: float`
   - `buffer_seconds: int = 60`
   - `is_expired() -> bool: return time.time() >= (self.expires_at - self.buffer_seconds)`

3. **WeComAPIError(Exception)**：`__init__(self, code: int, message: str)`
4. **WeComRateLimitError(WeComAPIError)**：用于 tenacity 重试

5. **WeComChannel(MessageChannel)**：
   - `channel_id = "wecom"`
   - `__init__`: 接受 `WeComConfig | dict | None`，初始化 `self._token: TokenInfo | None = None`，`self._http_client = httpx.AsyncClient(timeout=30.0)`
   - `_validate_config()`: 检查 corp_id/agent_id/secret 非空，否则抛 ChannelConfigError
   - `_get_access_token() -> str`: 检查缓存 → 调用 `GET https://qyapi.weixin.qq.com/cgi-bin/gettoken?corpid={corp_id}&corpsecret={secret}` → 解析 `access_token` + `expires_in` → 缓存为 TokenInfo(expires_at=time.time()+expires_in)
   - `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=10), retry=retry_if_exception_type(WeComRateLimitError), reraise=True)` 装饰 `_send_with_retry(self, payload: dict) -> dict`：
     - 获取 token，POST `https://qyapi.weixin.qq.com/cgi-bin/message/send?access_token={token}`
     - 检查 `errcode`：0=成功，45009=WeComRateLimitError，其他=ChannelSendError
   - `send_text(recipient, content) -> MessageResult`：
     - 解析 `recipient`：若含 `:` 则取冒号后的 userid，否则直接用
     - payload: `{"touser": userid, "msgtype": "text", "agentid": self.config.agent_id, "text": {"content": content}}`
     - 调用 `_send_with_retry`，返回 MessageResult
   - `send_markdown(recipient, content) -> MessageResult`：
     - 先尝试 `{"touser": userid, "msgtype": "markdown", "agentid": self.config.agent_id, "markdown": {"content": content}}`
     - 若 ChannelSendError 且 errcode 在 `{50001, 50002, 50003}` 或消息含 "not support"，回退 send_text
   - `supports_capture = True`（property）
   - `capture_openid(timeout=300) -> str`：
     - 打印引导提示（不用 Rich，直接 print）："请输入你的企业微信 userid\n（可在企业微信管理后台 → 通讯录 中查看）"
     - `userid = input("UserID: ").strip()`
     - 若为空则抛 ChannelError("userid 不能为空", channel_id=self.channel_id)
     - 返回 userid
   - `close()`: `await self._http_client.aclose()`
</action>

**3. 添加 `_do_wecom_capture(alias: str)` 辅助函数**（在 `_do_capture` 函数之后）：
```python
def _do_wecom_capture(alias: str) -> None:
    from claw_cron.channels import get_channel
    from claw_cron.channels.exceptions import ChannelError
    ch = get_channel("wecom")
    try:
        userid = asyncio.run(ch.capture_openid())
        console.print(f"[green]✓ UserID 已捕获: [bold]{userid}[/bold][/green]")
        save_contact(Contact(
            openid=userid,
            channel="wecom",
            alias=alias,
            created=datetime.now().isoformat(),
        ))
        console.print(f"[green]✓ 联系人已保存为 '{alias}'[/green]")
    except ChannelError as e:
        console.print(f"[red]Capture 失败: {e}[/red]")
```
