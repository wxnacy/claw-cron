## Summary

本次改动实现 v0.3.2 版本：将 contacts 存储键从单一 `alias` 改为 `channel/alias` 复合键，允许不同频道使用相同联系人别名而不互相覆盖。

## Changes

### Core: contacts.py

- 新增 `contact_key(channel, alias)` 辅助函数，生成 `channel/alias` 格式的复合键
- `save_contact()` 改用复合键存储，同名别名在不同频道不再互相覆盖
- `load_contacts()` 返回 `dict[channel/alias, Contact]`（键格式变更）
- `resolve_recipient()` 改为直接用 `contact_key(channel, alias)` 查找，逻辑更清晰
- 更新模块文档注释，示例 YAML 改为新键格式

### Commands: channels.py

- `list_contacts()` 展示时使用 `contact.alias`（而非 dict key）
- `delete` 命令新增 `--channel` 选项，支持同名别名多频道时精确删除
- 删除时若同名别名存在于多个频道且未指定 `--channel`，提示用户消歧义
- 导入新增 `contact_key`

### Commands: command.py / remind.py / update.py

- `channel_contacts` 字典构建方式统一改为 `{c.alias: c for c in contacts_data.values() if c.channel == ...}`，适配新键格式

### Version

- `__about__.py`: `0.3.1` → `0.3.2`

### Tests: test_contacts.py

- 新增 `TestContactKey` 测试类，验证复合键格式
- 更新 `TestLoadContacts` / `TestSaveContact` / `TestResolveRecipient` 中的断言，适配新键格式
- 新增跨频道同名别名测试用例（save 不覆盖、resolve 正确区分）
- 移除未使用的 `TemporaryDirectory` 导入

### Docs

- 删除 `docs/todo/claw-cron-v1.0.md`（已过期）
- 新增 `docs/todo/claw-cron-v0.1.md`（历史规划记录）
- 新增 `docs/todo/claw-cron-v0.3.md`（v0.3.x 规划，含 v0.3.2 需求）
